## 5. 前后依赖关系（编码的第一手输入）

### 5.1 依赖主线 1：<总开关字段> 驱动表单显隐与归一化

```
<总开关字段>
 ├─ <枚举1> → 隐藏 X，归一化 Y=null
 ├─ <枚举2> → 显示 X，<新建态/编辑态差异>
 └─ <枚举3> → ...
```

**新建态 vs 编辑态差异**（关键，最容易出 bug）：

| 场景 | 行为 | 置信度 |
|---|---|---|
| 新建态：A→B | ... | `[实现现状]` |
| 编辑态：后端返回 X | ... | `[实现现状]` |

**字段归一化规则**（提交时 mapper 统一处理）：
- <模式>：<字段> = null
- <条件>：清空 <字段>

### 5.2 依赖主线 2：<异步触发> 的触发与失效

```
<字段A> + <字段B> + <字段C> → 组成请求快照
  ├─ 任一变化 → 旧结果失效（递增 request id）
  ├─ 进入某步骤 → 自动触发
  └─ 成功 → ...；失败 → ...
```

### 5.3 依赖主线 3：<保存/提交> 的校验链

```
点击 Save
  ├─ ① 前端 schema 校验 → 失败跳最早错误步骤
  ├─ ② 后端校验接口
  │    ├─ errors → 阻塞
  │    ├─ warnings → 展示不阻塞
  │    └─ valid → 继续
  └─ ③ 创建/更新 → 成功/失败
```

**前端即时校验 vs 后端最终校验分工**：

| 层 | 拦截内容 |
|---|---|
| 前端即时 | <必填、格式、结构校验> |
| 后端最终 | <存在性、协议匹配、安全性、白名单> |
